開始把部落格發佈到GitHub吧!

前置作業

前一篇的文章大家有沒有仔細讀呀

在建構我們的部落格前須先完成Git的安裝及GitHub的註冊

如果都完成的話我們就來開始把部落格放上去吧!!

發佈開始! 🚀

Step1. 建立GitHub儲存庫

首先我們先登入GitHub

完成後我們就點擊右上角的「+」，點擊「New repository」

接著輸入Repository名稱

Repository命名規則
名稱一定要是{使用者帳號}.github.io

完成後就可以點擊「Create repository」，建立我們的儲存庫了

Step2. 打包Hugo專案

把我們本機上的專案整合起來吧!

首先先下Command移到放置網站的位置

再執行hugo將網站打包

1
2

CD 網站folder 
hugo  ## 打包網站

完成後就可以看到在網站資料夾下有一個public的資料夾

這就是我們要上傳到GitHub的檔案

接著就可以把檔案push到GitHub上了~~

Step3. 發佈到GitHub

接下來我們繼續下指令讓檔案上傳至GitHub上

首先我們先下Command移至public下

在做版本控制初始化

CD public  ## 移置public資料夾
git init  ## 版本控制初始化

在上傳檔案前我們要先建立一個遠端的數據庫

在此之前有一個小重點

因為GitHub的版本更新後已不使用登入GitHub做上傳檔案的驗證

已改為使用Personal access tokens做驗證

所以我們先來取得這個使用者存取token

到GitHub頁面又上方點選個人頭像選擇「Settings」

選擇頁面左下角的「Developer settings」

接著點選「Personal access tokens」再選擇「Tokens(classic)」

點選「Generate new token」再選擇「Generate new token (classic)」

接著就選擇此Token可使用的功能及Token過期時間

可以視自己所需要的功能勾選，筆者這裡只勾擇「repo」和「delete_repo」

勾選完後就可以點選「Generate token」

我們就會得到一組Token代碼，先把這組代碼保留下來等一下我們會使用到

完成上述步驟後我們終於可以來建立遠端的數據庫了

git remote add origin https://{剛剛的Token}.github.com/{使用者名稱}/{使用者名稱}.github.io

接下來我們就把檔案先加入暫存區，加入完後再執行commit把檔案提交到儲存庫中

git add .  ## 檔案先加入暫存區
git commit -m "[提交訊息]"  ## 檔案提交到儲存庫後面的提交訊息可任意修改

最後更新遠端儲存庫的檔案

git push -u origin master  ## 更新遠端儲存庫的檔案

接著就可以等待GitHub把網站發佈上去囉

差不多等個1-2分鐘網站就能順利發佈上去了

接著就可以前往https://{使用者名稱}.github.io/來看我們剛剛發佈的網站了

大功告成🙌
